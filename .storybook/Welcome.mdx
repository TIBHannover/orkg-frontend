# Introduction

Here we document some of the components used in the [ORKG Frontend](https://orkg.org). The scope of this component library includes:

- components that are reusable (i.e., they are made to be used more than once in the codebase)
- components that are usable in isolation (i.e., not relying on a global store)

The API endpoints from [sandbox.orkg.org](sandbox.orkg.org) are used in the stories to ensure the components are functional and display actual data. API endpoints are not mocked.

## HeroUI + Tailwind CSS v4

The ORKG frontend uses [Tailwind CSS v4](https://tailwindcss.com/) with [HeroUI v3](https://heroui.com/). We do not duplicate the HeroUI documentation here — refer to their docs for the component API. Always prefer HeroUI components over rolling your own; only build a custom component when the equivalent is missing or unsuitable.

The legacy wrappers under `src/components/Ui/` (originally created during the reactstrap → HeroUI migration) are deprecated. Import directly from `@heroui/react` in new code.

## Theming

Theme tokens are declared as CSS custom properties in [`src/app/globals.css`](https://gitlab.com/TIBHannover/orkg/orkg-frontend/-/blob/master/src/app/globals.css), grouped into:

- **HeroUI semantic tokens** (`--accent`, `--background`, `--surface`, `--foreground`, `--danger`, `--success`, `--warning`, `--muted`, `--border`, `--field-background`, etc.) under `:root` / `.light` and overridden under `.dark`. Reference these via the matching Tailwind utilities (`bg-surface`, `text-foreground`, `border-border`, …).
- **ORKG custom colors** under `@theme` (`--color-secondary`, `--color-secondary-darker`, `--color-dark`, `--color-smart`, `--color-smart-darker`, `--color-accent-darker`). Reference these as `bg-secondary`, `text-dark`, `bg-smart`, etc.
- **DataBrowser nesting levels** (`--db-level-0` … `--db-level-10`) for nested ComparisonTable / DataBrowser rows.
- **Layout token** `--max-width-container: 1320px` used for the page container width.

Both light and dark variants are defined; dark mode is activated via the `.dark` or `[data-theme='dark']` class.

The full token list (including dark-mode overrides and the `--db-level-*` ramp) lives in `src/app/globals.css`. Reference HeroUI tokens directly in JSX via Tailwind utilities — there is no `theme` object to consume.

```tsx
<div className="bg-surface text-foreground border border-border">
    <span className="text-muted">Hello</span>
</div>
```

Avoid hardcoded hex colors in components — always use the Tailwind utility tied to a token so light/dark theming continues to work.

## Styling conventions

- **Tailwind first.** Use Tailwind utilities for all new code.
- **`styled-components` is legacy** — present in older code but being phased out. Do not introduce new styled-components.
- **HeroUI compound components** (e.g. `Card.Header`, `Modal.Body`) — see the HeroUI v3 docs.
